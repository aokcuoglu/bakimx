# v0.1.1 — Manuel QA Kontrol Listesi

Bu doküman v0.1.1 sürümü için kapsamlı manuel test adımlarını içerir.

## Gereksinimler

- Çalışan bir PostgreSQL veritabanı
- `bun run dev` ile başlatılmış uygulama
- İki farklı tarayıcı (veya bir normal + bir gizli pencere)

---

## 1. Auth Kontrolleri

### 1.1 Kayıt
- [ ] `/register` sayfasına gidilir
- [ ] Tüm alanlar doldurulur ve kayıt olunur
- [ ] Başarılı kayıt sonrası `/app` dashboard'a yönlendirilir
- [ ] Workshop adı sidebar'da görünür

### 1.2 Giriş
- [ ] `/login` sayfasına gidilir
- [ ] Geçersiz e-posta ile giriş denenir → hata mesajı gösterilir
- [ ] Yanlış şifre ile giriş denenir → "E-posta adresi veya şifre hatalı" gösterilir
- [ ] Doğru bilgilerle giriş yapılır → `/app`'e yönlendirilir

### 1.3 Oturum Açıkken Auth Sayfaları
- [ ] Oturum açıkken `/login` sayfasına gidilir → `/app`'e yönlendirilir
- [ ] Oturum açıkken `/register` sayfasına gidilir → `/app`'e yönlendirilir

### 1.4 Çıkış
- [ ] Sidebar veya mobil menüden "Çıkış Yap" tıklanır
- [ ] `/login` sayfasına yönlendirilir
- [ ] `/app` sayfasına doğrudan gidilir → `/login`'e yönlendirilir

### 1.5 Yetkisiz API Erişimi
- [ ] Oturum kapalıyken tarayıcı konsolundan `/api/customers` POST isteği → 401
- [ ] Oturum kapalıyken `/api/vehicles` GET isteği → 401

## 2. Tenant İzolasyonu Kontrolleri

### 2.1 İki Workshop/User Oluşturma
- [ ] Workshop A + User A oluşturulur (örn. `testa@bakimx.com`)
- [ ] Workshop B + User B oluşturulur (örn. `testb@bakimx.com`)

### 2.2 Müşteri İzolasyonu
- [ ] User A ile Müşteri A1 oluşturulur
- [ ] User B ile Müşteri B1 oluşturulur
- [ ] User A, `/app/customers/[B1_id]` → 404 veya yönlendirme
- [ ] User B, `/app/customers/[A1_id]` → 404 veya yönlendirme

### 2.3 Araç İzolasyonu
- [ ] User A ile Araç A1 oluşturulur (Müşteri A1'e bağlı)
- [ ] User B ile Araç B1 oluşturulur
- [ ] User A, Araç B1 detayına erişemez

### 2.4 Kabul İzolasyonu
- [ ] User A ile Kabul A1 oluşturulur
- [ ] User B ile Kabul B1 oluşturulur
- [ ] User A, `/app/intakes/[B1_id]` → hata veya bulunamadı
- [ ] User B, `/app/intakes/[A1_id]` → hata veya bulunamadı

### 2.5 Servis Emri İzolasyonu
- [ ] User A, Kabul A1 için servis emri oluşturur
- [ ] User B, User A'nın servis emrine erişemez
