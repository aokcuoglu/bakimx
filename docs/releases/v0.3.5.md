# BakimX v0.3.5 — Digital Vehicle Service Passport

## Release Date
June 2026

## Summary
Vehicle-based digital service passport that shows a vehicle's complete service history, work orders, damages, photos, reminders, and provides a shareable customer-safe summary with QR code support.

## Features

### 1. Internal Vehicle Passport (`/app/vehicles/[id]/passport`)
- **Vehicle summary**: Plate, brand, model, year, mileage, VIN, fuel, transmission
- **Customer summary**: Name, phone, link to customer detail
- **Service history timeline**: Chronological timeline from all intake events
- **Work order history**: List of all service orders with status, totals, link to detail
- **Damage history**: All damage marks across all intakes, with severity badges
- **Photo history**: All photos across all intakes, with type labels
- **Maintenance reminders**: All reminders for the vehicle with status badges
- **Quick stats**: Count badges for intakes, orders, damages, photos, reminders
- **Share & QR section**: Create, manage, toggle, and delete passport tokens

### 2. Public Vehicle Passport (`/p/[token]`)
- **Customer-safe page**: No internal notes, raw OCR data, or internal IDs
- **Branded BakimX layout**: Workshop header, stats summary, section cards
- **Work order history**: With visibility controls
- **Damage history**: Zone labels, type labels, severity color badges
- **Photo checklist**: Labeled by type with dates
- **Maintenance reminders**: Filtered to exclude cancelled and internal notes
- **Service timeline**: Chronological event log
- **Workshop info**: Name, address, phone
- **Print/PDF button**: Browser print with print-optimized CSS
- **WhatsApp share text**: Pre-composed Turkish message
- **Copy link button**: Clipboard copy of public URL
- **Data safety notice**: Explains what's excluded

### 3. Visibility Controls
- **Create passport token**: Generates `nanoid(32)` token, unguessable URL
- **Activate/deactivate**: Toggle `isActive` on tokens
- **Optional expiry date**: Set `expiresAt` for time-limited sharing
- **Granular visibility toggles**: 
  - `showServiceHistory`
  - `showWorkOrders`
  - `showDamages`
  - `showPhotos`
  - `showReminders`
  - `showPaymentStatus`
- **Label**: Optional human-readable label for token management
- **Audit log**: All token create/update/delete actions logged via `AuditLogAction`

### 4. QR Code
- **QR code display**: Shown for each active passport token on internal page
- **Uses `qrcode.react`**: Lightweight, client-side QR generation
- **Download**: QR encodes the full public URL `/p/{token}`
- **No heavy dependency**: SVG-based, print-friendly

### 5. Security
- **Workshop-scoped queries**: All internal passport queries include `workshopId` filter
- **Token unguessable**: 32-character `nanoid` tokens
- **No cross-workshop leakage**: Public page validates token → vehicle → workshop chain
- **Data sanitization**: `sanitizePassportForPublic()` strips all internal data
- **Expired tokens**: `notFound()` for expired or inactive tokens

## API Routes

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/vehicles/[id]/passport` | Create passport token |
| PATCH | `/api/vehicles/[id]/passport/[tokenId]` | Update token (toggle, visibility) |
| DELETE | `/api/vehicles/[id]/passport/[tokenId]` | Delete token |
| GET | `/p/[token]` | Public passport page (no auth) |
| GET | `/p/[token]/pdf` | Printable HTML page (no auth) |

## Database Changes

### New Model: `VehiclePassportToken`
- `id` (cuid)
- `workshopId` (FK → Workshop)
- `vehicleId` (FK → Vehicle)
- `token` (unique, nanoid 32)
- `label` (optional)
- `isActive` (boolean, default true)
- `expiresAt` (optional DateTime)
- `showServiceHistory` (boolean, default true)
- `showWorkOrders` (boolean, default true)
- `showDamages` (boolean, default true)
- `showPhotos` (boolean, default true)
- `showReminders` (boolean, default true)
- `showPaymentStatus` (boolean, default false)
- `createdAt`, `updatedAt`
