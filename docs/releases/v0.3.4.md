# BakimX v0.3.4 — AI Service Advisor Lite

**Date:** 2026-06-12

## Summary

BakimX v0.3.4 introduces the AI Service Advisor — an intelligent assistant that analyzes customer complaints and vehicle data to suggest inspections, labor items, parts, customer-facing descriptions, and internal service notes. The advisor follows the same provider abstraction pattern as OCR (mock default, OpenAI, DeepSeek). AI suggestions are never auto-added; user confirmation is mandatory.

## Features

### 1. AI Provider Abstraction

- `AI_PROVIDER` env var: `mock` (default), `openai`, `deepseek`
- `AI_MODEL` env var: provider-independent model override
- `lib/advisor/types.ts` — `AiProvider` interface, `ServiceAdvisorInput`, `ServiceAdvisorResult`
- `lib/advisor/provider.ts` — Async factory with `getAdvisorProvider()` / `resetAdvisorProvider()`
- `lib/advisor/mock-advisor-provider.ts` — Keyword-matched Turkish demo suggestions for common complaints (brakes, oil, AC, engine, suspension)
- `lib/advisor/ai-advisor-providers.ts` — `OpenAiAdvisorProvider` and `DeepSeekAdvisorProvider` with structured JSON prompt and response parsing
- Missing API key returns clear Turkish error instead of silent fallback
- No hardcoded model names — defaults: `gpt-4o-mini` (OpenAI), `deepseek-chat` (DeepSeek)

### 2. Service Advisor Panel (Order Detail)

- Embedded in `/app/orders/[id]` after Complaint & Notes card
- Reads complaint, vehicle brand/model, mileage from order context
- Fetches previous work orders for same vehicle (up to 5) as additional context
- API: `POST /api/orders/advisor` — workshop-scoped, requires `intakeFormId`
- Displays:
  - **Önerilen Kontroller** — inspection checklist (checkbox-selected, not auto-added)
  - **İşçilikler & Parçalar** — labor and parts suggestions (checkbox-selected by default)
  - **Müşteri Açıklaması** — professional customer-facing description
  - **Servis İç Notu** — internal service note (not shown in customer output)
  - **Eksik Bilgi Uyarıları** — warnings for missing mileage, brand/model, etc.
- User actions for each suggested item:
  - Toggle selection (checkbox)
  - Edit inline (Düzenle)
  - Remove (Kaldır)
  - Add custom labor/part items
- Confirmation buttons:
  - **İş Emrine Ekle** — adds all selected labor/part items to order
  - **Yoksay** — dismisses AI suggestions

### 3. Standalone Service Advisor (New Order Page)

- Embedded in `/app/orders/new` left column
- Free-form input: complaint text, vehicle brand/model, mileage
- API: `POST /api/advisor` — workshop-scoped, no intakeFormId required
- Displays suggestions in read-only format (no order to add items to)
- Useful for pre-assessment before creating a work order

### 4. Intake Detail Integration

- `ServiceAdvisorPanel` added to intake detail info tab
- Provides AI suggestions during the intake/review phase
- On-add callback triggers page refresh

### 5. Safety & Privacy

- **AI sonucu tavsiye niteliğindedir** — disclaimer shown in every advisor panel
- **Kesin arıza teşhisi değildir** — explicitly stated
- `rawResponse` excluded from all API responses — never sent to client
- Raw AI response logged only server-side in development mode (`NODE_ENV !== "production"`)
- Workshop-scoped audit logs: `ai_advisor_requested` action with provider metadata
- AI suggestions must not auto-add items — explicit user confirmation required
- Provider label shown in UI (Demo (Mock), OpenAI, DeepSeek)

### 6. Mock Advisor Intelligence

- Keyword-matching for common Turkish automotive complaints:
  - `fren` → brake inspection, pad replacement, brake fluid
  - `yağ` → oil change, filter replacement, periodic maintenance
  - `klima` → AC gas check, cabin filter, compressor
  - `motor` → OBD scan, spark plugs, belt check
  - `süspansiyon` → shock absorber, ball joint, alignment
  - `ses`, `vites`, `sarsıntı` → appropriate fallbacks
  - Default → general vehicle check
- Missing info auto-warnings:
  - No mileage → "periyodik bakım önerileri eksik olabilir"
  - No brand/model → "parça önerileri spesifik olmayabilir"

## API Routes

### `POST /api/orders/advisor`

Request body:
```json
{
  "intakeFormId": "string (required)",
  "complaint": "string (optional, overrides intake complaint)",
  "vehicleBrand": "string (optional, overrides intake vehicle)",
  "vehicleModel": "string (optional)",
  "mileage": "number (optional)"
}
```

Response:
```json
{
  "success": true,
  "result": {
    "suggestedInspections": ["string"],
    "suggestedLabor": ["string"],
    "suggestedParts": ["string"],
    "customerDescription": "string",
    "internalNote": "string",
    "missingInfoWarnings": ["string"],
    "provider": "mock | openai | deepseek"
  }
}
```

### `POST /api/advisor`

Standalone advisor (no intake required).

Request body:
```json
{
  "complaint": "string (required)",
  "vehicleBrand": "string (optional)",
  "vehicleModel": "string (optional)",
  "mileage": "number (optional)"
}
```

Same response format as above.

## Known Limitations

1. `AI_PROVIDER` defaults to `mock` — returns keyword-matched demo data. Real AI requires `openai` or `deepseek` with API keys.
2. Mock advisor uses simple keyword matching; not contextually intelligent like LLM providers.
3. OpenAI/DeepSeek providers share API keys with OCR (`OPENAI_API_KEY`, `DEEPSEEK_API_KEY`).
4. `AI_MODEL` defaults: `gpt-4o-mini` (OpenAI), `deepseek-chat` (DeepSeek) — no hardcoded model names in non-mock providers.
5. AI suggestions are advisory only — not a definitive diagnosis.
6. `rawResponse` is stored only in memory (provider result), never persisted to DB or sent to client.
7. Previous work orders context limited to 5 most recent for the same vehicle.
8. Standalone advisor has no "add to order" action — read-only preview.
9. Currency is TRY only.
10. No real SMS/WhatsApp/payment/invoice integration (all mocks or client-side links).
