# BakimX v0.1.5 — Müşteri Yönetimi UX Alignment

**Release Date:** June 2026
**Previous Release:** v0.1.4 — İş Emri UX Alignment

---

## Summary

v0.1.5 redesigns and hardens the BakimX customer management experience so the product feels like a professional auto service operations tool. The customer list becomes a benchmark-style table on desktop and a card grid on mobile with type/tag filters, “Yeni Müşteri” / “Bakiye Özeti” / “Excel İçe Aktar” CTAs. The new customer form supports an individual/corporate toggle with a side panel for tags, source, price group, discount, risk note, and communication permissions (WhatsApp / SMS / E-posta + KVKK approval date). The customer detail page is rebuilt into a structured view (header, communication permissions, balance summary foundation, vehicles, work orders, intake forms, notes, risk banner, safe delete). A new `/app/customers/balances` page provides a foundation-level balance summary KPI + list derived from order totals (no real accounting / payment module claim). The Customer model is safely extended with type, contact, identity, tag, source, price group, discount, risk, communication permissions and KVKK approval fields — fully backward compatible. Two new placeholder affordances are stubbed (Excel import, voice fill) and clearly marked “Yakında”. No Docker, no real OCR, no real SMS, no real payment module, no e-fatura, no WhatsApp Business API.

## Changes

### 1. Customer Schema Extension (`prisma/schema.prisma`)

Backward-compatible schema additions on `Customer`:

- `type: CustomerType` enum (default `individual`) — individual | corporate
- `firstName`, `lastName` made **optional** (existing rows unaffected, action layer derives a displayable name)
- `fullName`, `companyName`, `contactName` (optional)
- `phone2`, `city`, `district`, `address` (optional)
- `identityNumber`, `taxNumber`, `taxOffice` (optional)
- `notes` (optional, internal only)
- `tag: CustomerTag` enum — standard (default) | vip | risky | fleet
- `source: CustomerSource` enum — referral | google | social_media | walk_in | existing | other
- `priceGroup: CustomerPriceGroup` enum — standard (default) | discounted | fleet
- `discountRate: Float?` (default 0)
- `riskNote: String?`
- `whatsappConsent`, `smsConsent`, `emailConsent` boolean (default false)
- `kvkkApprovedAt: DateTime?`
- New composite indexes: `(workshopId, type)`, `(workshopId, phone)` for faster list filtering

New shared constants in `src/lib/constants.ts`:

- `CUSTOMER_TYPES` (label + color), `CUSTOMER_TAGS`, `CUSTOMER_SOURCES`, `CUSTOMER_PRICE_GROUPS`

### 2. Validation (`src/lib/validation.ts`)

- `customerCreateSchema` rewritten with a `superRefine`:
  - `type: "individual"` → requires `firstName` or `fullName`; `lastName` or `fullName`; `phone`
  - `type: "corporate"` → requires `companyName`; `phone`
- `email` validated if provided
- `discountRate` 0–100 range
- All error messages in Turkish

### 3. Customer Server Actions (`src/app/app/customers/actions.ts`)

- `createCustomerAction` — type-aware; persists all new fields, normalizes phones, parses KVKK date
- `updateCustomerAction(customerId, formData)` — same data flow, scoped by `workshopId`
- `deleteCustomerAction(customerId)` — safe delete: returns friendly error if vehicles/intakes exist; audit-logged
- `searchCustomersAction(query)` — searches name, company, contact, full name, phone, email, plate
- `listCustomersForWorkshop(workshopId, filter)` — supports text + type + tag + source filters with plate search

All mutations use `requireAuth` + `workshopId`-scoped Prisma queries; no exposure of other tenants' data.

### 4. Customer List UX (`src/app/app/customers/page.tsx`, `src/components/app/customer-list.tsx`)

- Page title “Müşteriler” + breadcrumb “Ana Panel / Müşteriler”
- Header CTAs:
  - **+ Yeni Müşteri** (primary)
  - **Bakiye Özeti** (secondary)
  - **Excel İçe Aktar** (placeholder, disabled, with tooltip “Excel içe aktarma yakında.”)
- Info banner about Excel import / payment module status
- Search input “Ad, telefon, e-posta veya plaka ara…” + type filter (Tüm Tipler / Bireysel / Kurumsal) + tag filter + clear filter
- Desktop table (≥lg): Müşteri / Telefon / Tip / Etiket / Araç / İş Emri / Toplam İşlem / Kayıt Tarihi / İşlem
- Mobile cards (<lg): name + type badge + tag badge, phone, email, vehicle/work-order/total mini-stats
- Empty states for no data and no search results
- Each row uses initials / corporate icon avatar and direct `tel:` link

### 5. New Customer Form (`src/app/app/customers/new/page.tsx`, `src/components/app/customer-create-form.tsx`)

- **Bireysel / Kurumsal** segmented toggle (desktop and mobile)
- Main form: Temel Bilgiler (Ad Soyad or Şirket Adı + Yetkili, Telefon, Telefon 2, E-posta, İl, İlçe, Adres)
- Side panel (desktop) / below form (mobile):
  - **Müşteri Profili**: Etiket (Standart / VIP / Riskli / Filo), Müşteri Kaynağı (Seçilmedi / Tavsiye / Google / Sosyal Medya / Yoldan Geldi / Mevcut Müşteri / Diğer), Fiyat Grubu (Standart / İndirimli / Filo), İndirim %, Risk / Uyarı Notu
  - **İletişim İzinleri**: WhatsApp izni, SMS izni, E-posta izni, KVKK Onay Tarihi
  - **Vergi / Kimlik Bilgileri**: TC Kimlik No, Vergi No, Vergi Dairesi
- “Sesle Doldur” placeholder button (disabled, “Yakında”)
- Mobile sticky save bar
- Turkish validation messages, no silent failures

### 6. Customer Detail Polish (`src/app/app/customers/[id]/page.tsx`, `src/components/app/customer-detail.tsx`)

- Header: avatar (initials / building icon), name, type/tag badges, phone, email, city
- Quick actions: Düzenle, Yeni İş Emri, WhatsApp (placeholder), Geri
- Bakiye Özeti card: KPI grid (İş Emri, Yapılan İş, Genel Toplam, Bizim Alacağımız) + disclaimer banner
- Vehicles list with PlateBadge, link to vehicle detail
- Work orders list (with status, payment status, totals, estimated delivery)
- Intake forms without an order (context)
- Notes section
- Right sidebar: Müşteri Özeti (tag, source, price group, registration date), İletişim İzinleri (read-only), Vergi / Adres
- Risk note banner (red)
- Safe delete (with `DeleteCustomerButton` confirmation)
- `?edit=1` query param opens edit mode directly

### 7. Balance Summary Foundation (`src/app/app/customers/balances/page.tsx`, `src/lib/customer-totals.ts`)

- New helper `summarizeCustomerOrders(orders, lastActivityAt)` calculates per-customer: ordersCount, workDone, grandTotal, paid, partial, unpaid, remaining, customerCredit, hasOverdue
- New page `/app/customers/balances`:
  - 6 KPI cards: Müşteri Sayısı, Yapılan İş, Bizim Alacağımız, Toplam Tahsilat, Müşteri Alacağı, Geciken Tahsilat
  - Filter bar (search + type)
  - Desktop table: Müşteri / Telefon / Araç / İş Emri / Borç / Tahsilat / Bizim Alacak / Müşteri Alacağı / Son İşlem / Durum
  - Mobile cards with key stats
  - Disclaimer banner: “Tahsilat modülü henüz aktif değildir. Bu sayfa iş emri toplamlarına göre ön hazırlık amaçlıdır; gerçek muhasebe verisi göstermez.”
- No new payment/billing module. No legal accounting claims.

### 8. Search Improvements

- Customer list searches: firstName, lastName, fullName, companyName, contactName, phone, phone2, email, plate (via related vehicles)
- Vehicle list search now also matches customer `fullName` and `companyName`
- Order list still searches plate, customer name, work order no, phone
- All searches are tenant-scoped (workshopId)

### 9. UI/Design Consistency

- BakimX palette preserved: primary navy `#0B1F3A`, primary blue `#2563EB`, accent sky `#38BDF8`, soft background `#F8FAFC`, deep sidebar `#0F172A`
- New components: `CustomerTypeBadge`, `CustomerTagBadge`, `PriceGroupBadge` (consistent rounded-pill badges with semantic colors)
- No orange/amber main styling, no noisy gradients
- PlateBadge / StatusBadge / PaymentBadge reused from v0.1.4
- No “ServisCep” references
- All Turkish labels follow the new terminology: Müşteriler, Yeni Müşteri, Müşteri Listesi, Bireysel, Kurumsal, Telefon, Araç, İş Emri, Toplam İşlem, Kayıt Tarihi, Bakiye Özeti, İletişim İzinleri, KVKK Onay Tarihi, Risk / Uyarı Notu

### 10. UX Helpers

- `lib/format.ts` adds `customerDisplayName(customer)` — null-safe display name for individual/corporate/legacy records
- All places that rendered `customer.firstName + customer.lastName` now use the helper or an equivalent inline fallback
- No regressions in old work order list, intake list, vehicle list, public output, PDF

### 11. Tenant Isolation & Security

- Every new Prisma query is `workshopId`-scoped
- Mutations verify ownership (`prisma.customer.findFirst({ where: { id, workshopId } })`)
- Delete action refuses to delete customers with linked vehicles / intakes
- Audit log entries: `customer_created`, `customer_updated`, `customer_deleted`
- No secrets, no new env vars, no .env files committed
- No Docker

### 12. Placeholder Affordances

- **Excel İçe Aktar** (button) — disabled, tooltip “Excel içe aktarma yakında.” (no implementation)
- **Sesle Doldur** (button) — disabled, copy “Sesle doldurma özelliği yakında.” (no implementation, no speech-to-text)
- **WhatsApp paylaş** (button in detail page) — disabled, copy “WhatsApp paylaşımı yakında” (no WhatsApp Business API)

### 13. Documentation & Seed

- README updated to v0.1.5
- `prisma/seed.ts` extended to create a corporate demo customer (Anadolu Lojistik A.Ş.) with all new fields populated
- New docs: `docs/releases/v0.1.5.md`, `docs/QA/v0.1.5-manual-checklist.md`

## Known Limitations

- Tahsilat / ödeme modülü sadece etiket; Bakiye Özeti iş emri toplamlarından türetilir
- “Excel İçe Aktar” ve “Sesle Doldur” yalnızca UI placeholder
- WhatsApp paylaşımı yalnızca UI placeholder (WhatsApp Business API yok)
- Placeholder modüller (Teklifler, Randevular, Bakım Hatırlatmaları, Stok / Parçalar, Tedarikçiler, Kasa, Raporlar) hâlâ “Yakında”
- Plaka tanıma / sesle doldurma / barkod yalnızca placeholder
- Gerçek SMS, gerçek OCR, gerçek ödeme, e-fatura/e-arşiv yok
- Çok şubeli kurumsal modül yok
- S3 depolama sağlayıcısı henüz uygulanmadı (placeholder)
- HEIC dosya desteği yok
- @react-pdf/renderer sunucu PDF üretimi hâlâ aktif değil
