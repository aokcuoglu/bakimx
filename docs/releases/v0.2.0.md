# BakimX v0.2.0 — Dashboard & Operations Overview

**Release Date:** June 2026
**Previous Release:** v0.1.5 — Müşteri Yönetimi UX Alignment

---

## Summary

v0.2.0 redesigns the BakimX protected dashboard (`/app`) from a simple welcome page into a professional operations overview for auto repair shops. Workshop owners and service advisors can now immediately understand active work orders, today's deliveries, pending approvals, missing photos, overdue orders, recent customer activity, and weekly operational trends — all from a single screen.

The dashboard features KPI cards with real counts, alert banners for critical items, CSS-based weekly operations and status distribution charts, a full active work orders table (desktop) / card list (mobile), and five right-side widgets covering today's deliveries, waiting approvals, missing photos, recent customers, and quick actions. All data is tenant-scoped, uses real Prisma queries, and gracefully handles empty states.

No Docker, no real SMS, no real OCR, no real payment module, no WhatsApp Business API. No new chart libraries — charts are pure CSS/SVG zero-dependency.

## Changes

### 1. Dashboard Data Helpers (`src/lib/dashboard/queries.ts`)

New server-side helper module with eight functions, all scoped by `workshopId`:

- `getDashboardStats(workshopId)` — returns 6 KPI counts in a single parallelized query batch:
  - `activeOrders`: not delivered/cancelled
  - `todayDeliveries`: `estimatedDeliveryAt` is today, not delivered/cancelled
  - `waitingApprovals`: status = `waiting_approval`
  - `missingPhotoIntakes`: intakes missing one or more required photo types (front, rear, left_side, right_side, dashboard_mileage)
  - `overdueDeliveries`: `estimatedDeliveryAt` before today, not delivered/cancelled
  - `lastWeekOrders`: created in the last 7 days
- `getActiveWorkOrders(workshopId, limit)` — latest active orders with customer, vehicle, photo completion, totals
- `getTodayDeliveries(workshopId)` — deliveries due today
- `getWaitingApprovals(workshopId)` — orders in `waiting_approval` status
- `getMissingPhotoItems(workshopId)` — active orders with missing required photos
- `getRecentCustomers(workshopId, limit)` — latest customers
- `getWeeklyOperations(workshopId)` — orders created per day for last 7 days
- `getWorkStatusDistribution(workshopId)` — current month order status distribution

All helpers use safe date boundaries (`todayStart`, `daysAgo`, `monthStart`), handle null `estimatedDeliveryAt`, and fall back gracefully when no data exists.

### 2. Dashboard Page Redesign (`src/app/app/page.tsx`)

- **Widgets row** (3-col desktop): Today Deliveries, Waiting Approvals, Missing Photos

### 4. Missing Photos Logic

The dashboard computes missing required photos by:
- Defining required photo types: `front`, `rear`, `left_side`, `right_side`, `dashboard_mileage`
- For each active intake, counting how many required types have at least one photo uploaded
- If the count < 5 (all required types present), it is flagged as missing photos

This is a best-effort safe computation. The KPI card counts intakes with missing photos; the widget lists specific active orders with missing counts.

**Known limitation**: The computation checks for at least one photo of each required type. If a photo exists for a type but is corrupted or unusable, it is still counted as present. This is documented in the limitations section.

### 5. Charts (Zero Dependencies)

Both chart components use pure CSS/SVG — no external chart library was added:

- **Weekly operations**: flex-based vertical bar chart with Turkish day labels (Paz–Cmt), value labels on top of bars, responsive height
- **Status distribution**: horizontal stacked bar segments with color legend, percentage labels, and category counts

Empty states show "Son 7 günde iş emri bulunmuyor." / "Bu ay iş emri bulunmuyor." respectively.

### 6. Quick Actions

Six shortcut cards in a 2x3 grid:
- **Yeni İş Emri** → `/app/orders/new`
- **Yeni Müşteri** → `/app/customers/new`
- **Yeni Araç** → `/app/vehicles/new`
- **Onay Bekleyenler** → `/app/orders?status=waiting_approval`
- **Eksik Fotoğraflar** → `/app/intakes`
- **Müşteri Bakiye Özeti** → `/app/customers/balances`

All routes are existing and functional. No "coming soon" routes are used.

### 7. Global Search

The topbar global search (already in AppShell) remains unchanged. It searches by plate, customer name/company, phone, and work order number via `/app/orders?q=...`. No changes needed — existing behavior supports all required search dimensions.

### 8. Tenant Isolation

Every dashboard query is scoped by `workshopId` obtained from the authenticated session. No internal IDs are exposed to the browser. All 8 query helpers receive `workshopId` as their first parameter. The `getAppData()` auth check remains the gatekeeper for the entire `/app/*` route.

## Known Limitations

- **Missing photos computation**: Checks for at least one photo per required type. Corrupted/ unusable photos are still counted as present. No HEIC detection at dashboard level (HEIC is already rejected at upload).
- **Approval status tracking**: The dashboard shows intake approval status on active work orders. If an intake is `waiting_approval` and there is no ServiceOrder, it appears in the KPI but not in the active work orders list (which only queries ServiceOrder records). This is consistent with the data model.
- **Charts are CSS-based**: No interactive features (tooltips, zoom, export). Pure visual summary.
- **Alert banner is static**: Alerts link to relevant list pages but do not support dismiss/acknowledge. No notification system.
- **Today deliveries use server timezone**: Date boundaries are computed in the server's local timezone. This is acceptable for early-stage single-timezone operations.
- **No real-time updates**: The dashboard is a server-rendered page. Data reflects the state at page load. No WebSocket or polling.
- **Global search navigates to `/app/orders?q=...`**: No dedicated search results page. Results appear in the orders list with current filters applied.
- **Placeholder modules remain placeholder**: Quotes, appointments, reminders, inventory, suppliers, cash, reports are still "Yakında".
- **No Docker, no real SMS, no real OCR, no WhatsApp Business API** (unchanged from v0.1.x).
- **S3 storage provider still not implemented** (unchanged).
- **@react-pdf/renderer server PDF generation still not active** (unchanged).
