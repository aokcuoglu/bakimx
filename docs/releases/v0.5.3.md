# BakimX v0.5.3 ŌĆö Workshop Settings & Business Profile

**Release Date:** June 2026

## Overview

This release introduces a comprehensive Workshop Settings page with 7 tabs: Business Profile, Branding, Communication, Working Hours, Appointment Rules, PDF Templates, and Security. Workshop owners can now manage all operational configuration from a single unified interface. Additionally, this release includes rate limiting for settings API endpoints, audit logging for all setting changes, enhanced login security, and branding support in public-facing portals and PDFs.

## New Features

### 1. Settings Page (`/app/settings`)

### 2. WorkshopSettings Model

New `WorkshopSettings` model in Prisma schema with fields for:

- Branding: `pdfLogoUrl`, `publicPortalLogoUrl`, `passportLogoUrl`, `themeColor`, `accentColor`
- Communication: `smsProvider`, `smsApiKey`, `smsSenderName`, `whatsappProvider`, `whatsappApiKey`, `whatsappPhoneNumber`, `emailProvider`, `emailApiKey`, `emailFromAddress`, `emailFromName`
- Working Hours: `weekdayStart`, `weekdayEnd`, `weekdayWorkingDays`, `weekendStart`, `weekendEnd`, `weekendWorkingDays`, `holidayEnabled`, `holidayDates`
- Appointment Rules: `defaultAppointmentDuration`, `bufferDuration`, `reminderTimings`
- PDF Templates: `workOrderTemplate`, `servicePassportTemplate`, `collectionReceiptTemplate`

### 3. Settings API Endpoints

- `GET /api/settings` ŌĆö Fetch all workshop settings
- `POST /api/settings/branding` ŌĆö Update branding (logo URLs, theme colors)
- `POST /api/settings/communication` ŌĆö Update communication providers
- `POST /api/settings/working-hours` ŌĆö Update working hours
- `POST /api/settings/appointment-rules` ŌĆö Update appointment rules
- `POST /api/settings/pdf-templates` ŌĆö Update PDF templates

All endpoints include rate limiting (30 requests/minute per IP per endpoint).

### 4. Validation Schemas

New Zod validation schemas added to `src/lib/validation.ts`:

- `businessProfileSchema` ŌĆö Validates business profile fields
- `brandingSchema` ŌĆö Validates logo URLs and hex color codes
- `communicationSettingsSchema` ŌĆö Validates provider selections and API keys
- `workingHoursSchema` ŌĆö Validates time formats (HH:mm) and working days
- `appointmentRulesSchema` ŌĆö Validates duration (5-480min) and buffer (0-120min)
- `pdfTemplateSchema` ŌĆö Validates template content

### 5. Rate Limiting

New `src/lib/rate-limit.ts` ŌĆö In-memory rate limiter:

- 30 requests per 60-second window per key
- Applied to all settings API endpoints
- Returns 429 with `Retry-After` header when exceeded

### 6. Enhanced Login Security

- Login API route now includes rate limiting
- Login action uses validated schema
- Failed login attempts are tracked

### 7. Branding in Public Portals & PDFs

- Public share pages (`/p/[token]`, `/s/[token]`) now use workshop logo and branding from settings
- PDF generation routes now reference `pdfLogoUrl` and `passportLogoUrl` from WorkshopSettings

### 8. Workshop Form Enhancement

- Workshop onboarding form (`/app/workshop`) updated with additional fields matching business profile schema

## Database Changes

New model `WorkshopSettings`:
- 1:1 relation with `Workshop` (via `workshopId` unique field)
- Auto-created on first settings access via `getWorkshopSettings()`
- All fields nullable or have defaults, safe for initial creation

Run `bunx prisma db push` to apply schema changes.

## Bug Fixes (Pre-Release)

- **S1:** Workshop update API now handles all business profile fields (district, email, website, taxNumber, taxOffice, invoiceTitle)
- **S2:** Settings page auto-creates `WorkshopSettings` row if missing (no 404 on first visit)
- **S3:** Rate limiting added to login and all settings endpoints to prevent brute-force attacks
- **S4:** Public portals and PDFs now properly fall back to workshop logo when settings logos are not set

## Breaking Changes

None. All new features are additive. The `/app/workshop` update API now accepts additional fields but maintains backward compatibility.

## Limitations

- Communication settings (SMS, WhatsApp, Email) use mock providers by default ŌĆö real provider integration requires API key configuration
- Rate limiter is in-memory only ŌĆö resets on server restart (suitable for single-instance deployments)
- PDF templates use Handlebars-like syntax but are stored as raw text ŌĆö rendering engine not yet implemented
- No file upload for logos ŌĆö URLs must be provided directly
- No Docker configuration in this project
- No secrets committed to repository
