# BakimX v0.5.2 — Billing & Collections Enhancement

**Release Date:** June 2026

## Overview

This release enhances the billing and collections module with UX improvements, receivable aging reports, payment timeline enhancements, cancellation reasons, collection reminders, and export foundations.

## New Features

### 1. Cashbox UX Polish (`/app/cashbox`)

- **Date presets** in payment list: Today, Yesterday, This Week, Last Week, This Month, Last Month, Last 30/90 days, Custom range
- **Daily collection chart** on cashbox dashboard (last 14 days)
- **Yaşlandırma (Aging)** link from cashbox sidebar
- **Export buttons** (CSV + Print) on cashbox dashboard
- **Improved filters** in payment list page with custom date range support

### 2. Customer Receivable Aging (`/app/cashbox/aging`)

New page showing receivable aging buckets:
- **0-7 gün** — Recent receivables (emerald)
- **8-30 gün** — Aging receivables (amber)
- **31-60 gün** — Overdue receivables (orange)
- **60+ gün** — Critically overdue receivables (rose)

Each bucket shows total amount, customer count, and customer-level breakdown with name, phone, and amount.

### 3. Work Order Payment Timeline

Enhanced `PaymentHistoryCard` in order detail view:
- Shows **completed** and **cancelled** collections in the timeline
- Cancelled entries display with strikethrough amount, red styling, and cancellation reason
- **İptal Edilen** (cancelled amount) summary row
- **Send Reminder** button when there's an outstanding balance

### 4. Collection Reminder Foundation

- **"Tahsilat hatırlatması gönder"** action on order detail (when balance > 0)
- Uses existing `notifyCollectionReminder` communication trigger (reuses `payment_reminder` template)
- Respects customer communication permissions (smsConsent, whatsappConsent, emailConsent)
- Audit log created for each reminder sent
- Mock providers by default (no real SMS/WhatsApp/Email unless configured)

### 5. Payment Cancellation Reason

- **Required reason** when cancelling a collection payment
- `cancellationReason` field added to `CollectionPayment` model
- Cancel button now shows a textarea for reason input (mandatory)
- Cancellation reason displayed on payment detail page
- Audit log includes reason and original amount in metadata

### 6. Export Foundation

- **CSV export** endpoint at `/api/cashbox/export` supporting:
  - `type=collections` — Tahsilatlar CSV
  - `type=receivables` — Alacaklar CSV
  - `type=aging` — Yaşlandırma CSV
- Print view uses `PrintButton` client component (not inline `window.print()` in server components)
- CSV files include BOM for proper Turkish character display in Excel

## Database Changes

New field on `CollectionPayment`:
- `cancellationReason String?` — Reason for cancellation (null if not cancelled)

Run `bunx prisma db push` to apply schema changes.

## Bug Fixes (Pre-Release)

- **C1:** Fixed `totalPaid` calculation in order detail including cancelled collections — now filters to `completed` only
- **C2:** Fixed `notifyVehiclePassportShare` using wrong template key (`payment_reminder` → `vehicle_passport_share`)
- **C3:** Fixed `window.print()` in server components — extracted `PrintButton` client component
- **W5:** Fixed empty `customerName` in `SendReminderButton` — now passes customer display name from order data

## Breaking Changes

None. All new features are additive. The `cancelCollectionAction` now **requires** a reason parameter (previously defaulted to "Kullanıcı iptali"). The API route passes reason, but direct server action calls must also provide a reason.

## Limitations

- Payment link is a placeholder only — no real payment gateway integration
- No invoice/e-archive generation
- No Docker configuration
- Reminder uses the existing `payment_reminder` template (no dedicated `collection_reminder` template yet)
- Aging buckets are based on order creation date, not invoice due date (no invoice system)
- CSV export does not support pagination beyond 5000 rows
