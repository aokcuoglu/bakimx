# BakimX v0.3.0 — Tahsilat / Kasa Foundation

## Summary
BakimX v0.3.0 introduces the operational cashbox and collection foundation for workshops. This release adds manual payment records, partial payment support, work order payment status connection, customer balance improvements, collection history and dashboard receivable visibility.

## Included
- Kasa / Tahsilat module foundation
- Cashbox overview page
- Manual payment record creation
- Payment list and detail pages
- Payment cancellation without deleting records
- Work order payment status connection
- Partial payment support
- Full payment support
- Overpayment visibility
- Customer balance upgrade
- Collection history
- Revenue/receivable summary cards
- Dashboard cash/receivable widget
- Payment method and payment status badges
- README/release notes/QA checklist updates
- Local development with bun/npm
- No Docker local workflow

## Foundation-only scope
- This is operational collection tracking only
- It is not a legal accounting system
- It is not an invoice/e-archive/e-invoice system
- It does not process online payments
- It does not integrate with banks or POS systems

## Not included
- Docker
- Real payment gateway
- Credit card processing
- Bank/POS integration
- Accounting module
- E-invoice/e-archive/e-fatura
- Tax declaration/accounting export
- Real SMS/WhatsApp/email sending
- Multi-branch enterprise management

## Database Migration Required

This release adds a new `CollectionPayment` table and extends `ServiceOrder` with payment fields. **You must run the database migration before deploying:**

```bash
# Option 1: Prisma migrate (recommended)
npx prisma migrate deploy

# Option 2: If setting up fresh
npx prisma migrate dev --name add_collection_payment

# Option 3: If using db push (development only)
npx prisma db push
```

The migration file is at: `prisma/migrations/20260610120000_add_collection_payment/migration.sql`

Changes:
- New `CollectionPayment` table with all fields and indexes
- New columns on `ServiceOrder`: `paidAmount` (Float, nullable), `remainingAmount` (Float, nullable), `lastPaymentAt` (DateTime, nullable)
- New enum `PaymentMethod` (cash, credit_card, bank_transfer, other)
- New enum `CollectionStatus` (completed, cancelled, refunded)
- New enum value `overpaid` added to `PaymentStatus`

## Known Limitations
1. No real payment gateway integration (intentional)
2. No online payment processing (intentional)
3. No e-invoice/e-archive generation (out of scope)
4. No official accounting integration (out of scope)
5. No bank/POS integration (out of scope)
6. No real SMS/WhatsApp for payment receipts (mock only)
7. Monthly collection trend chart uses simple CSS/SVG bars (no heavy analytics)
8. Overdue logic uses estimatedDeliveryAt before today + remaining > 0 (documented limitation)
9. Customer balance computed dynamically (no stored snapshot)
10. Currency is TRY only
