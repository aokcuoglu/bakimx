# v0.2.5 — Tedarikçiler Foundation

**Release Date:** June 2026

## Overview

v0.2.5 introduces the supplier management foundation for BakimX, enabling auto repair shops to manage their supplier contacts, link suppliers to parts, and track critical stock by supplier.

## New Features

### Supplier Module
- **Supplier list** (`/app/suppliers`) — search, filter, KPI cards, desktop table + mobile cards
- **New supplier** (`/app/suppliers/new`) — multi-section form with validation
- **Edit supplier** (`/app/suppliers/[id]/edit`) — full edit form
- **Supplier detail** (`/app/suppliers/[id]`) — summary, operational info, related parts, critical stock, purchase placeholders

### Supplier Contact Fields
- Tedarikçi Adı (required)
- Yetkili Kişi, Telefon, Telefon 2, E-posta, Web Sitesi
- İl, Adres
- Vergi No, Vergi Dairesi
- Kategori

### Supplier Operational Fields
- Ortalama Teslimat Süresi (gün)
- Ödeme Vadesi (gün)
- Performans Notu
- Dahili Notlar
- Durum (Aktif / Pasif)

### Supplier Related Parts
- Parts linked via `supplierId` on `PartStockItem`
- Related parts displayed on supplier detail
- Critical stock parts from supplier highlighted
- Link to part detail from supplier page

### Parts Module Integration
- Supplier selector in new/edit part forms
- Linked supplier displayed on part detail and list
- `supplierName` text fallback preserved for backward compatibility
- Parts search includes linked supplier name
- Supplier selector scoped by `workshopId`

### Dashboard Integration
- "Kritik Stok Tedarikçileri" widget on dashboard
- Shows suppliers with parts in critical/out-of-stock status
- Links to supplier detail

### Navigation
- Tedarikçiler active in sidebar (no "Yakında" badge)
- Removed from `COMING_SOON_PREFIXES`

### Purchase/Request Placeholders
- "Teklif İste" button (disabled, "Yakında" badge)
- "Satın Alma Talebi Oluştur" button (disabled, "Yakında" badge)
- Helper text: "Satın alma ve teklif modülleri ilerleyen sürümlerde eklenecektir."

## Data Model

### New: Supplier
- `id`, `workshopId`, `name`, `contactPerson?`, `phone?`, `phone2?`, `email?`, `website?`, `city?`, `address?`, `taxNumber?`, `taxOffice?`, `category?`, `paymentTermDays?`, `averageDeliveryDays?`, `performanceNote?`, `internalNote?`, `isActive`, `createdAt`, `updatedAt`
- Indexes: `workshopId`, `name`, `phone`, `email`, `city`, `isActive`

### Updated: PartStockItem
- Added `supplierId?` (optional relation to Supplier)
- Added `@@index([supplierId])`
- `supplierName` and `supplierPhone` preserved for backward compatibility

## Not Included
- No supplier API integration
- No accounting/invoice module
- No purchase order workflow
- No real SMS/WhatsApp/email sending
- No e-commerce product feed
- No multi-branch enterprise module
- No Docker
- No external supplier API

## Backward Compatibility
- Parts with `supplierName` text only continue to work
- `supplierId` is optional on all parts
- Existing parts forms still show `supplierName`/`supplierPhone` fields
- No destructive migration required

## Security
- All supplier queries scoped by `workshopId`
- `updateMany` with compound where for workshop-scoped updates
- Cannot access another workshop's supplier by direct URL
- Cannot attach another workshop's supplier to a part
- Audit logging on all supplier mutations
