# BakimX v0.1.2 — Public Output & PDF Foundation

**Release Date:** June 2026
**Previous Release:** v0.1.1 — MVP Hardening & UX Polish

---

## Summary

v0.1.2 focuses on making BakimX's customer-facing output page professional enough for real workshop demos, establishing PDF/print output foundation, adding WhatsApp share link generation, and improving service order totals across the platform.

---

## Changes

### 1. Public Output Page Polish (`/s/[token]`)

- **Professional BakimX navy/blue layout** with mobile-first design
- **Print-friendly CSS structure** with `print:` prefixed classes
- **Workshop header** showing name, city, address, phone, and logo
- **Customer & vehicle summary** with plate, brand, model, year, mileage, VIN
- **Legal disclaimer** section: "Bu çıktı, servis kabul ve işlem özeti amacıyla oluşturulmuştur."
- **No internal IDs exposed** — all public data is stripped of internal database IDs
- **No `internalNote` displayed** to customers
- **Invalid/expired token shows branded error page** (not-found.tsx)
- **Photo checklist** shows "Fotoğraf önizlemesi depolama entegrasyonu sonrası gösterilecektir." when fileUrl is missing
- **Service order totals** broken down by parts/labor with grand total
- **Actions section** with print, WhatsApp share, and copy link buttons

### 2. PDF / Print Foundation

- **New route:** `/s/[token]/pdf` — server-rendered HTML optimized for print-to-PDF
- **@react-pdf/renderer** installed as dependency for future server-side PDF generation
- **`components/pdf/public-output-document.tsx`** — React PDF document component (foundation)
- **Fallback:** Browser print with "Yazdır / PDF Olarak Kaydet" and "Yazdırılabilir Sayfa" buttons
- **PDF HTML route** generates a clean, print-optimized A4 HTML rendering
- Legal disclaimer included in PDF output
- No external IDs in PDF output

### 3. WhatsApp Share Link Generation

- **New module:** `lib/share/whatsapp.ts`
- `generateWhatsAppShareText()` — generates Turkish WhatsApp share text with public link and optional total amount
- `getWhatsAppShareUrl()` — creates `wa.me` URL with encoded text
- **WhatsApp share button** on public output page (green "WhatsApp ile Paylaş" button)
- **WhatsApp share button** in intake detail approval section (internal user)
- Share text format: "Merhaba, aracınızın BakimX servis kabul ve işlem özetine aşağıdaki bağlantıdan ulaşabilirsiniz: {link}"
- Optionally includes total: "Toplam tutar: ₺..."
- No WhatsApp Business API integration — manual share link generation only

### 4. Better Service Order Totals

- **New utility module:** `lib/totals.ts`
  - `calculateLineTotal()` — safely calculates line total handling null prices
  - `formatLineTotal()` — formats with "Fiyat girilmedi" for missing prices
  - `calculateGroupTotal()` — calculates group subtotal (parts/labor/all)
  - `formatOrderSummary()` — returns formatted parts/labor/grand totals with `hasAnyPrice` flag
- **Updated across all locations:**
  - `intake-detail.tsx` — order tab now shows "Fiyat girilmedi" for items without prices
  - `order-detail.tsx` — uses centralized utility, shows "Fiyat girilmedi"
  - `public-share-page.tsx` — uses centralized utility for parts/labor breakdown
  - `/s/[token]/pdf` route — uses same calculations in PDF output
- **TRY formatting** with `Intl.NumberFormat` via existing `formatTRY()`
- **Null/undefined price handling** — safe calculations, no NaN

### 5. Customer & Vehicle Search Improvements

- **Customer search** (`/app/customers?q=`) already supports:
  - Search by first name (case-insensitive)
  - Search by last name (case-insensitive)
  - Search by phone
  - Server-side filtering with workshopId scope
  - Client-side filtering via `CustomerList` component
  - Empty state when no results
  - Mobile-friendly search input (h-12)
- **Vehicle search** (`/app/vehicles?q=`) already supports:
  - Search by plate (case-insensitive)
  - Search by brand (case-insensitive)
  - Search by model (case-insensitive)
  - Search by customer first/last name
  - Empty state when no results
  - Mobile-friendly search input

### 6. Image Storage Provider Preparation

- **Storage abstraction layer** (`lib/storage/`) extended:
  - `types.ts` — `StorageProvider` interface with `StorageUploadResult`
  - `mock-storage-provider.ts` — unchanged, safe for local dev
  - `storage-provider.ts` — factory with `STORAGE_PROVIDER` env var support (throws clear errors for unimplemented providers)
  - **NEW:** `supabase-storage-provider.ts` — placeholder with TODOs
  - **NEW:** `s3-storage-provider.ts` — placeholder with TODOs
  - **NEW:** `index.ts` — barrel export
- No storage env vars required
- Public output page handles missing `fileUrl` gracefully with placeholder text

### 7. Image Lint Warning Cleanup

- **2 `<img>` ESLint warnings in intake-detail.tsx** fixed
- Added `eslint-disable-next-line @next/next/no-img-element` comments for `blob:` object URL previews (not suitable for `next/image`)
- Both locations: photo capture preview (line 547) and `PhotoChecklistItem` preview (line 1073)
- Lint is now clean (0 errors, 0 warnings)

### 8. Next.js Middleware / Proxy Migration Status

- **Assessment:** Current middleware (`src/middleware.ts`) is working correctly with Next.js 16
- **Build warning detected:** "The 'middleware' file convention is deprecated. Please use 'proxy' instead."
- **Impact:** Middleware still functions correctly, but the convention is deprecated in Next.js 16
- **Action:** Documented as pending framework migration item. Not performing risky auth refactor in v0.1.2.
- **Future:** Will need to migrate from `middleware.ts` to Next.js 16 "proxy" convention in a future release
- `getSession()` uses `cookies()` from `next/headers` with proper `await`
- Public paths, public prefixes, API protection, app auth redirect all functional

### 9. Public Output Data Safety

- **Server component strips all internal IDs** before passing to client
  - No workshop `id` exposed
  - No intake/customer/vehicle/order IDs exposed
  - No damage mark or photo IDs in public data
  - No `internalNote` passed to client
  - No `email` exposed (removed from public data)
  - Customer phone intentionally shown (workshop context)
- **Invalid/inactive tokens** return `notFound()` which renders branded error page
- **No stack traces** in public routes
- **No password/session data** in public output

### 10. UX Polish

- **BakimX blue/navy identity** consistently applied:
  - Primary navy `#0B1F3A`
  - Primary blue `#2563EB` (PDF link, accent buttons)
  - Accent sky `#38BDF8`
  - Soft background `#F8FAFC`
  - Deep background `#0F172A`
- WhatsApp share button uses `#25D366`
- Print-friendly styles with `print:` prefixed classes
- Mobile-friendly spacing and typography
- Copy link functionality with "Kopyalandı!" feedback

## Known Limitations

- Real SMS provider not integrated (mock/demo mode)
- Real OCR/VIN extraction not implemented
- Real image storage provider not integrated (abstraction ready, mock in use)
- WhatsApp Business API not integrated (manual share link only)
- No payment/billing module
- No multi-branch enterprise management
- `@react-pdf/renderer` installed but not yet generating server-side PDF
  - Current PDF output is print-optimized HTML via `/s/[token]/pdf`
  - React PDF document component is foundation for future use
- No Docker files (by design)
