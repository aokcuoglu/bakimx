# BakımX v0.5.7 — Production Security & Release Blockers

**Status:** Release candidate
**Type:** Security hardening (no new features, no schema/migration, no Docker changes)
**Scope:** Closes the P0 production blockers from the engineering security audit, plus the v0.5.7.1 follow-up patch.

---

## Summary

v0.5.7 closes six P0 release blockers and a small follow-up patch (v0.5.7.1). No new product features were added; no database schema or migration changes were made; no Docker/local-dev changes. All changes are server-side guards, public-output hardening, and supporting helpers.

## Security fixes

### 1. Public registration removed
- `/register` now redirects to `/login` (no registration form/flow).
- `POST /api/auth/register` returns **404** and no longer creates accounts.
- `registerAction` deleted; `middleware.ts` no longer treats `/register` as a special public path.
- No other `Workshop`/`User` creation path exists in the codebase (verified).
- Login and seed/demo accounts (`demo@bakimx.com`) unaffected.

### 2. Login hardening
- **Rate limiting:** per-client-IP, 8 attempts / 60s, applied *before* any DB work; returns **429** with `Retry-After`.
- **Account-enumeration defense:** unknown e-mail and wrong password return the **same** Turkish message (`E-posta adresi veya şifre hatalı`) and both run a bcrypt comparison (real hash or a fixed dummy hash) to equalize timing.
- **Session rotation:** `session.destroy()` + set identity + `session.save()` on login (clears any pre-existing/fixated session). Cookie flags unchanged (`httpOnly`, `secure` in prod, `sameSite=lax`).
- Shared core in `src/lib/auth-login.ts` used by both the API route and the (otherwise dead) `loginAction`.

### 3. Cron endpoint hardening
- `CRON_SECRET` is **required in all environments**: missing config → **500**; missing/incorrect Bearer token → **401** (constant-time `timingSafeEqual`).
- `.env.example` and `README.md` updated.

### 4. Status transition guards
- `src/lib/status-transitions.ts` adds enum whitelists + transition maps mirroring the UI.
- **Intake `approved` is reachable only through the customer OTP flow** (`verifyOtpAction`); the generic status action always rejects a manual transition to `approved` → customer-approval/OTP cannot be bypassed.
- Order transitions validated against the UI transition map; `paymentStatus` validated against the enum. Arbitrary/garbage status values are rejected.

### 5. Public share / PDF output escaping
- `/s/[token]/pdf` and `/p/[token]/pdf` HTML-escape **all** user/workshop-controlled fields (`escapeIntakeForHtml` / `escapePassportForHtml` at the data layer + inline escaping of workshop/vehicle/customer/logo/custom-template) → stored-XSS surface closed.
- React pages (`/s/[token]`, `/p/[token]`) continue to rely on JSX auto-escaping (no double-escaping).

### 6. Work order number uniqueness
- `generateWorkOrderNo` gained a random suffix (no same-millisecond collisions).
- `generateUniqueWorkOrderNo` performs a workshop-scoped DB check with retry; applied across all three order-creation paths (orders, quote→order, appointment→order).
- Durable fix (`@@unique([workshopId, workOrderNo])`) **deferred and documented** — it requires a backfill migration on existing data and is intentionally not part of this release.

### v0.5.7.1 follow-up patch
- **Public intake timeline** now filters internal-only events (`internal_note_added`, `labor_session_started`, `labor_session_stopped`), mirroring the passport sanitizer.
- **Pricing visibility consistency:** `showOrderItems` controls item visibility; `showPaymentStatus` controls prices, line totals, grand total, and the WhatsApp total — applied at the sanitizer source so `/s/[token]`, `/s/[token]/pdf`, and the public share component behave consistently. The passport already gated prices by `showPaymentStatus`.
- **`showPhotos`** is enforced on the `/s/[token]/photos/[photoId]` byte route (in addition to the sanitizers).
- **app-shell lint fix:** the sidebar-restore `setState`-in-effect was replaced with a `useSyncExternalStore`-backed localStorage store (SSR-safe, no hydration mismatch, no `eslint-disable`).
- **Docs:** README note that nginx must set the real client IP via `X-Forwarded-For` (login rate-limit depends on it).

## Behavior changes (intentional)

- A share/passport with `showPaymentStatus` **off** now hides item prices, line totals, grand total, and the WhatsApp total (items still visible under `showOrderItems`).
- Internal-note and labor-session timeline events never appear on the public intake share.

## Deferred / out of scope (see v0.5.8 backlog)

- Money fields stored as `Float` (precision).
- `prisma.$transaction` around multi-write flows (cashbox/order/OCR).
- Shared/durable rate-limit store (currently in-memory per process).
- OTP entropy/expiry/attempt-limit hardening.
- `@@unique([workshopId, workOrderNo])` durable constraint (needs backfill migration).
- Pre-existing `db.ts` placeholder DB fallback.
