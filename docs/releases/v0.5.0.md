# BakimX v0.5.0 — Real Communication Integrations

**Release Date:** June 2026

## Overview

This release adds real communication infrastructure for SMS, WhatsApp, and Email notifications across the BakimX platform. Communication providers are fully abstracted behind a provider interface, and the app works without any configured providers (defaults to mock mode).

## New Features

### 1. Communication Provider Architecture (`src/lib/communications/`)

Abstracted provider system supporting hot-swappable providers via environment variables:

- **SMS:** `MockSMSProvider` (default), `NetgsmProvider`
- **WhatsApp:** `MockWhatsAppProvider` (default), `WhatsAppBusinessProvider` (foundation)
- **Email:** `MockEmailProvider` (default), `ResendProvider`

Environment variables:
- `SMS_PROVIDER=mock|netgsm` (default: mock)
- `WHATSAPP_PROVIDER=mock|business` (default: mock)
- `EMAIL_PROVIDER=mock|resend` (default: mock)

### 2. Notification Center (`/app/settings/notifications`)

Manage per-channel notification templates:
- **SMS templates** — Appointment Reminder, Intake Approval, Quote Ready, Work Order Completed, Maintenance Reminder, Payment Reminder, Vehicle Passport Share
- **WhatsApp templates** — Same as SMS with richer formatting
- **Email templates** — HTML email templates for each notification type

Templates are per-workshop and use variable interpolation (`{customerName}`, `{vehiclePlate}`, etc.).

### 3. Appointment Notifications

- Automatic notification on appointment creation
- Configurable reminder triggers (24h before, 1h before)

### 4. Intake Approval Notifications

- SMS and WhatsApp link sent when intake approval is requested
- Reuses existing approval flow

### 5. Quote Notifications

- Customer notified when quote status changes to "sent"
- Quote approval link included in notifications

### 6. Work Order Completion Notifications

- Vehicle ready notification on `ready_for_delivery` status
- Customer portal link included
- Payment reminder on delivery if payment is pending

### 7. Maintenance Reminder Automation

- Upcoming/overdue reminder notifications via preferred channel
- Integrated with existing reminder module

### 8. Communication Log (`/app/communications`)

Full log of all communications:
- Type (SMS, WhatsApp, Email)
- Provider
- Recipient
- Status (sent, failed, pending)
- Related entity
- Timestamps

### 9. Customer Communication Preferences

- Per-customer consent toggles: SMS, WhatsApp, Email
- System respects preferences before sending
- KVKK approval timestamp tracked

### 10. Security

- No secrets exposed in templates or logs
- Workshop-scoped communication logs
- Template sanitization (strips `<script>`, event handlers, `javascript:` URIs)
- Rate limiting foundations (per-workshop per-customer per-channel, 30/min default)

### 11. Reminder Scheduling

- **Cron endpoint** (`GET/POST /api/cron/reminders`) — Scheduled job for appointment and maintenance reminder processing
- **Appointment reminder processing** — Finds appointments within 24h/1h, sends reminders, updates `reminderStatus` and `lastRemindedAt`
- **Maintenance reminder processing** — Finds due/upcoming reminders with `preferredChannel != 'none'`, sends notifications, updates `reminderStatus` and `lastRemindedAt`
- **Deduplication** — `lastRemindedAt` field on both Appointment and MaintenanceReminder prevents duplicate sends
- **Authentication** — `CRON_SECRET` environment variable required in production; Bearer token authentication

## Database Changes

New enums:
- `CommunicationType` (sms, whatsapp, email)
- `CommunicationStatus` (pending, sent, failed)

New models:
- `CommunicationTemplate` — per-workshop notification templates with unique constraint on (workshopId, templateKey, channel)
- `CommunicationLog` — full communication audit log with workshop scoping and entity relations

Run `bunx prisma db push` to apply schema changes.

## Breaking Changes

None. All new features are additive and default to mock providers that require no external configuration.

## Limitations

- Mock providers log to console only; real providers require API keys
- WhatsApp Business API provider is a foundation (template message API not yet implemented)
- Appointment reminder scheduling requires an external cron job (e.g., Vercel Cron, Upstash QStash, or manual call to `/api/cron/reminders`)
- Rate limiting is in-memory (not distributed); for multi-instance deployments, use Redis-backed rate limiting
