# BakimX v0.4.0 — Technician Mobile Workspace

## Release Date
June 2026

## Overview
v0.4.0 introduces a dedicated mobile-first technician workspace, enabling assigned technicians to manage work orders, upload evidence, request parts, and complete repair tasks directly from their phones.

## New Features

### 1. Technician Model
- New `Technician` entity with fields: `id`, `workshopId`, `fullName`, `phone`, `role`, `isActive`, `createdAt`, `updatedAt`
- Roles: Usta, Teknisyen, Servis Danışmanı, Yönetici
- All records workshop-scoped
- Management UI in Workshop Settings page

### 2. Work Order Assignment
- New fields on `ServiceOrder`: `assignedTechnicianId`, `assignedAt`, `completedAt`
- Technician assignment from Order Detail page (manager view)
- Technician name shown on order cards and detail pages
- One-click assign/unassign with timeline event logging

### 3. Technician Dashboard (`/app/technician`)
- Mobile-first card layout
- KPI cards: Assigned, In Progress, Waiting, Completed, Today's Delivery
- Sections: Active Jobs, Waiting Jobs, Recently Completed
- Technician selector dropdown for manager view
- Checklist progress bars on order cards

### 4. Technician Work Order View (`/app/technician/orders/[id]`)
- Vehicle summary, customer summary, complaint display
- Checklist system with check/uncheck and progress bar
- Repair photos display (before/during/after)
- Internal notes (workshop-only, never public)
- Parts request with status flow (Requested → Prepared → Delivered)
- Labor time tracking (start/stop sessions with duration)
- Sticky bottom action bar: Start Work → Hold → Complete
- Status flow: Bekliyor → İşlemde → Tamamlandı

### 5. Technician Checklist
- `ChecklistItem` model with categories: Inspection, Repair, Delivery
- Per-item check/uncheck with timestamps
- Notes per checklist item
- Add/delete custom checklist items
- Progress bar visualization

### 6. Repair Photos
- Extended `VehiclePhoto` with `serviceOrderId` and `phase` field
- Phases: intake (before), repair_progress (during), delivery (after)
- Photos linked to Vehicle, Work Order, and Vehicle Passport

### 7. Internal Notes
- `InternalNote` model for workshop-only communication
- Not visible on public portal, passport, or customer outputs
- Add/delete notes from technician order view

### 8. Parts Request Flow
- `PartsRequest` model with statuses: requested, prepared, delivered
- Status progression buttons in technician view
- Timeline events logged on status changes

### 9. Labor Time Tracking
- `LaborSession` model with start/end/duration
- Start/stop buttons in technician view
- Duration auto-calculated in minutes
- Dashboard-ready for future analytics

### 10. Mobile UX
- Large touch targets throughout
- Bottom sticky action bar on order detail
- Camera-first workflow design
- Responsive layouts for all screen sizes

### 11. Vehicle Passport Integration
- Completed work orders automatically feed into Vehicle Service Passport
- Repair photos included in passport photo history
- Technician assignment shown in work order details on passport
- Internal notes filtered from public passport (security)

### 12. Manager Dashboard Widget
- "Usta İş Durumu" widget on main dashboard
- Shows per-technician: active jobs, in-progress count, delayed count
- Links to Technician Dashboard for details

### 13. Security
- All queries workshop-scoped
- No cross-workshop technician access
- Internal notes never exposed on public pages
- Technician uploads protected by workshop scope
- Passport sanitization filters internal event types

## Database Changes
- New models: `Technician`, `ChecklistItem`, `InternalNote`, `PartsRequest`, `LaborSession`
- New enums: `TechnicianRole`, `ChecklistCategory`, `PartsRequestStatus`
- New fields on `ServiceOrder`: `assignedTechnicianId`, `assignedAt`, `completedAt`
- New field on `VehiclePhoto`: `serviceOrderId`
- New relations on `Workshop`: technicians, checklistItems, internalNotes, partsRequests, laborSessions

## Breaking Changes
- None. All changes are additive.

## Migration Required
- Run `prisma db push` or `prisma migrate dev` to apply schema changes
- Run `bunx prisma generate` to regenerate the Prisma client
- Optionally run `bunx tsx prisma/seed.ts` to add sample technicians
