# BakimX v0.5.4 — UX Alignment & Operations Speed

**Release Date:** June 2026

## Overview

This release focuses on aligning UX patterns across all list pages and accelerating daily operations. Every major list view — orders, vehicles, customers, quotes, appointments — now shares a consistent structure: KPI summary cards at the top, sticky-column action menus on desktop, mobile-optimized card layouts, and Turkish plate badges. Dropdown action menus replace inline action links, reducing visual clutter while providing quick access to view, edit, create work orders, schedule appointments, and archive operations.

## New Features

### 1. KPI Cards (`kpi-cards.tsx`)

New reusable `KpiCards` and `KpiCardsLinked` components:

- Responsive grid layout (2-col mobile, 4/6-col desktop)
- Accent-colored count badges per metric
- Optional link behavior — clicking a KPI card filters the list
- Active state ring highlight for the currently selected filter
- `KpiCardsLinked` auto-generates filter URLs from `baseHref` and `filterValue`

### 2. Actions Menu (`actions-menu.tsx`)

New `ActionsMenu` and `MobileActionsMenu` dropdown components:

- Built on `@base-ui/react` `DropdownMenu` primitives (shadcn/ui base-nova style)
- Contextual action links: View, Edit, Create Work Order, Create Appointment, View Passport, Archive
- All props optional — only relevant actions appear
- Touch-friendly `size-7`/`size-8` trigger buttons
- Sticky right-column alignment on desktop tables

### 3. Quick Actions (`quick-actions.tsx`)

New `QuickActions` component and factory functions:

- `orderQuickActions()`, `vehicleQuickActions()`, `customerQuickActions()`, `appointmentQuickActions()`, `quoteQuickActions()`
- Primary (blue) and secondary (outlined) action variants
- Responsive — icon-only on mobile, icon + label on desktop

### 4. Turkish Plate Badge (`plate-badge.tsx`)

Redesigned plate badge component:

- Authentic Turkish plate layout: blue city-code band, white letter band, blue number band
- `formatTurkishPlate()` utility splits raw plate into city code / letters / numbers
- Consistent `h-8` height across all contexts
- Linked plates navigate to vehicle detail pages

### 5. Enhanced Status Badges (`status-badge.tsx`)

- `StatusBadge` and `PaymentBadge` now support `size` prop (`"sm"` | `"md"`)
- Re-export `PlateBadge` from `status-badge.tsx` for convenience

### 6. Dropdown Menu Primitive (`dropdown-menu.tsx`)

New shadcn/ui dropdown menu component based on `@base-ui/react` Menu primitives:

- Full API: `DropdownMenu`, `DropdownMenuTrigger`, `DropdownMenuContent`, `DropdownMenuItem`, `DropdownMenuSeparator`, `DropdownMenuGroup`, `DropdownMenuLabel`, `DropdownMenuCheckboxItem`, `DropdownMenuRadioGroup`, `DropdownMenuRadioItem`, `DropdownMenuSub`, `DropdownMenuSubTrigger`, `DropdownMenuSubContent`, `DropdownMenuPortal`, `DropdownMenuShortcut`
- Proper portal positioning with `MenuPrimitive.Positioner`
- Slide-in/fade animations, ring focus styles

## Bug Fixes

- **B1:** Plate badges now render consistently across all list views with proper Turkish format (city code / letters / numbers)
- **B2:** Mobile card layouts no longer overflow horizontally — all cards use `min-w-0 flex-1` pattern
- **B3:** Dropdown menus use proper `@base-ui/react` API (not Radix) for shadcn/ui base-nova compatibility
- **B4:** Sticky action columns on desktop tables use `group-hover:bg-slate-50/60` for proper hover state
- **B5:** Customer list filter state preserved across re-renders

## Breaking Changes

None. All changes are additive. The `PlateBadge` import path has been preserved via re-export from `status-badge.tsx`.

## Limitations

- No Docker configuration in this project
- No database schema changes in this release
- Quick actions component created but not yet integrated into page headers (available for future use)
- KPI cards for vehicles (Documents Expiring, Service Due) show conditionally — values are 0 until tracking features are implemented
